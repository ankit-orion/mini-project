Section 01

Problem Formulation

A manufacturing factory produces three products — A, B, and C. Each product consumes a different combination of three shared resources: machine hours, labor hours, and raw material units. The goal is to decide how many units of each product to produce per week to maximize total profit while staying within resource capacities.

Decision Variables

Let:

x₁ = units of Product A produced
x₂ = units of Product B produced
x₃ = units of Product C produced

All variables must be non-negative (cannot produce negative units).

Dataset: Resource Usage Table

ResourceProduct AProduct BProduct CCapacity
Machine Hours2 hrs1 hr3 hrs240 hrs
Labor Hours1 hr2 hrs2 hrs200 hrs
Raw Material4 kg3 kg2 kg360 kg
Profit/Unit$50$40$60—

Mathematical Model

Objective Function — Maximize Profit
Z = 50x₁ + 40x₂ + 60x₃

Subject To (Constraints)
2x₁ + x₂ + 3x₃ ≤ 240 (Machine hours capacity)
x₁ + 2x₂ + 2x₃ ≤ 200 (Labor hours capacity)
4x₁ + 3x₂ + 2x₃ ≤ 360 (Raw material capacity)
x₁, x₂, x₃ ≥ 0 (Non-negativity)
Section 02

Python Implementation

The model is solved using scipy.optimize.linprog with the HiGHS solver — the industry-standard open-source LP solver. Since linprog minimizes by default, the profit coefficients are negated to convert the maximization problem.

Python · optimization.py
# ─────────────────────────────────────────────────
#  Mini Project 5: Production Optimization
#  Tool: scipy.optimize.linprog (HiGHS solver)
# ─────────────────────────────────────────────────

from scipy.optimize import linprog
import numpy as np

# ── Objective: Maximize Profit
#    linprog minimizes, so negate coefficients
c = [-50, -40, -60]          # Profit: A=$50, B=$40, C=$60

# ── Inequality Constraints: A_ub @ x ≤ b_ub
A_ub = [
    [2, 1, 3],   # Machine hours per unit
    [1, 2, 2],   # Labor hours per unit
    [4, 3, 2],   # Raw material (kg) per unit
]
b_ub = [240, 200, 360]        # Capacities

# ── Non-negativity bounds
bounds = [(0, None)] * 3

# ── Solve
result = linprog(c, A_ub=A_ub, b_ub=b_ub,
                  bounds=bounds, method='highs')

# ── Extract solution
x = result.x
max_profit = -result.fun

print(f"Product A: {x[0]:.2f} units")
print(f"Product B: {x[1]:.2f} units")
print(f"Product C: {x[2]:.2f} units")
print(f"Maximum Profit: ${max_profit:.2f}")

Code Walkthrough

Negate Objective Coefficients

linprog only minimizes. By negating the profit vector [50, 40, 60] → [−50, −40, −60], minimizing the negative is mathematically equivalent to maximizing the original profit.

Define Constraint Matrix A_ub

Each row of A_ub represents one resource. Column values represent how much of that resource one unit of each product consumes. The solver ensures A_ub @ x ≤ b_ub.

Set Non-Negativity Bounds

Production quantities cannot be negative. bounds = [(0, None)] * 3 constrains each variable to be ≥ 0, with no upper limit.

Run HiGHS Solver

The HiGHS solver uses the simplex or interior-point method to traverse the feasible polytope and identify the optimal vertex (corner point) that satisfies all constraints while maximizing profit.

Section 04

Simulation Experiment Design

To understand how robust the solution is, we run 7 simulation scenarios — each modifying one or more resource constraints. This sensitivity analysis reveals which resource is the most critical bottleneck.

Python · simulation.py
# ─────────────────────────────────────────────────
#  Simulation: Vary constraints across 7 scenarios
# ─────────────────────────────────────────────────

scenarios = [
    {"name": "Base Case",         "machine": 240, "labor": 200, "material": 360},
    {"name": "Machine +25%",      "machine": 300, "labor": 200, "material": 360},
    {"name": "Labor -20%",        "machine": 240, "labor": 160, "material": 360},
    {"name": "Material +50%",     "machine": 240, "labor": 200, "material": 540},
    {"name": "All +30%",          "machine": 312, "labor": 260, "material": 468},
    {"name": "Machine -30%",      "machine": 168, "labor": 200, "material": 360},
    {"name": "Tight Budget",      "machine": 180, "labor": 150, "material": 270},
]

results = []
for s in scenarios:
    b = [s["machine"], s["labor"], s["material"]]
    r = linprog(c, A_ub=A_ub, b_ub=b, bounds=bounds, method='highs')
    results.append({
        "scenario": s["name"],
        "A": r.x[0], "B": r.x[1], "C": r.x[2],
        "profit": -r.fun
    })

Scenario Design Rationale

📈 Machine +25%

Simulates adding a new machine shift or purchasing additional equipment. Tests if machine hours is the primary bottleneck.

📉 Labor −20%

Simulates a workforce reduction (layoffs, strike, or seasonal shortage). Measures profit sensitivity to labor cuts.

🏭 Material +50%

Simulates a bulk supply deal or warehouse expansion. Extreme surplus to see if production can scale without machine/labor.

🚀 All Resources +30%

Simulates full-factory expansion — scaled up all three resources proportionally to see best-case growth.

⚠️ Machine −30%

Simulates equipment breakdown or maintenance downtime. High-stress test on the most constrained resource.

💰 Tight Budget

All resources reduced to 75% of base. Represents an austerity quarter or budget-constrained operation.

Section 06

Sensitivity Analysis

Sensitivity analysis quantifies how much profit changes per unit increase in each resource — this is the shadow price (dual variable) of each constraint.

Shadow Price · Machine Hours
~$20.77/hr

Machine +25% added 60 hrs → profit gained $738. Each extra machine hour worth ~$20.77 in additional profit.

Shadow Price · Labor Hours
~$6.92/hr

Labor −20% cut 40 hrs → profit dropped $277. Labor hours have roughly 1/3 the value of machine hours.

Shadow Price · Raw Material
~$4.44/kg

Material +50% added 180 kg → profit gained $800. Lower per-unit value but large volume available.

Most Valuable Resource: Machine hours have the highest shadow price. Investing in additional machine capacity (new equipment, extended shifts) yields the highest marginal profit return.
Least Sensitive Resource: Raw material has the lowest shadow price — even a 50% surplus only yields $800 extra because machine and labor constraints become the new bottlenecks.
Biggest Risk: Machine breakdown (−30% scenario) caused a 14.8% profit drop — larger than the gain from a 25% increase. Downside risk is asymmetric and must be hedged (maintenance contracts, spare parts, redundancy).
Section 07

Final Summary of Insights

🎯

Optimal Solution

Producing 40 units each of Products A, B, and C yields the maximum weekly profit of $6,000 under base constraints — with zero slack in any resource.

🔩

Machine Capacity is King

The machine hours constraint has the highest shadow price (~$20.77/hr). Every new machine hour added is worth more than a labor hour or material unit. Expansion priority: machines first.

📦

Product Mix is Dynamic

The optimal production mix is not fixed. When machines are tight, produce more B (least machine-intensive). When machines are abundant, shift to C (highest profit at $60/unit).

📈

Proportional Expansion Works

Expanding all resources by 30% scales profit by exactly 30% ($6k → $7.8k). The LP is homogeneous — this confirms no diminishing returns in the feasible zone.

⚠️

Downside Risk is Steep

A 30% machine loss causes a 14.8% profit drop — not 30%, because labor can partially substitute. But the mix becomes suboptimal (B-heavy), making recovery sensitive to market conditions.

💡

Material Surplus is Wasteful

Even with 50% more raw material, the solver couldn't use it all (533/540 used) because machine and labor limits bind first. Buying excess raw material without machine expansion is inefficient capital allocation.

Recommendations

Invest in Machine Capacity

Shadow price analysis shows each additional machine hour is worth ~$20.77 in profit. Adding a second shift or leasing equipment should be evaluated against this ROI benchmark.

Maintain a Dynamic Production Plan

Use LP optimization weekly rather than maintaining a fixed production mix. The optimal mix changes with every resource fluctuation — what's optimal in base conditions is suboptimal when machines fail.

Build Machine Redundancy

Given the asymmetric downside risk of machine failure (−14.8% profit from a 30% capacity loss), invest in preventive maintenance contracts and cross-trained labor to reduce downtime exposure.

Right-Size Material Procurement

Never purchase raw material beyond what machine and labor capacity can process. The simulation shows material surplus beyond ~375 kg generates zero additional profit under base conditions.
